# Game Designer Agent Role

## 목적

이 프로젝트의 AI는 자동 플레이어가 아니라 `게임 기획자 역할의 에이전트`로 둔다. 플레이 중 보드 상태를 읽고, 현재 난이도 감각과 다음 튜닝 액션을 제안하는 것이 1차 목표다.

## 현재 입력

- 보드 크기
- 타일 종류 수
- 가능한 시작 체인 수
- 가장 긴 같은 색상 연결 크기
- 현재 점수
- 사용한 이동 수

## 현재 출력

- 난이도 판정: `Easy`, `Normal`, `Hard`
- 보드 진단: 가능한 시작점과 최장 체인
- 기획 의도: 지금 보드를 어떤 용도로 볼지
- 리스크: 플레이어가 막히거나 너무 쉽게 느낄 가능성
- 다음 액션: 타일 종류, 목표 점수, 이동 수, 연출 강화 방향

## 왜 이렇게 쓰는가

포코팡 스타일 퍼즐의 재미는 단순히 블록이 터지는 것보다 `연결할 선택지가 잘 보이는지`, `3개 이상 연결하는 보상이 충분히 느껴지는지`, `다음 레벨에서 무엇을 조정해야 하는지`에 달려 있다.

그래서 첫 에이전트는 플레이를 대신하는 AI보다, 보드 밸런스를 읽고 기획 코멘트를 주는 deterministic designer agent로 시작한다. 이후 LLM 기반 기획 에이전트를 붙이더라도 같은 입력/출력 계약을 유지하면 확장하기 쉽다.
