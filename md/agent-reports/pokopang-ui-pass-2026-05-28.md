# Pokopang-Style UI Pass - 2026-05-28

## Goal

Improve the prototype HUD toward the readable structure of a PokoPang-style mobile puzzle battle while keeping the project visually original.

## Source Read

- PokoPang public descriptions and screenshots show a combat target/context above the puzzle board.
- The core interaction remains a compact colored block board, with score/time/boost feedback visible during play.
- This project should not copy original assets, names, or exact visual treatment.

## Applied Direction

- Keep score, timer, and fever charge in a compact top information strip.
- Treat the fever charge as a play-state meter, not a rainbow-bomb meter.
- Add a battle lane below the top strip with enemy or boss badge on the left and HP bar on the right.
- Keep boss HP as a strong red combat read, while regular enemies use a cooler blue read.
- Shorten score text so the HUD survives narrow mobile widths.

## Portfolio Fit

This change strengthens the portfolio goal by making the playable prototype read more like a puzzle-combat system instead of a generic board test scene. It also preserves the AI designer angle: score, fever, enemy HP, and feedback are clearer surfaces for the agent to analyze.
