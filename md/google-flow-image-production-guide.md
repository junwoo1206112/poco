# Google Flow 이미지 제작 가이드

작성일: 2026-05-28

## 목적

이 문서는 Poko Puzzle 프로토타입의 적, 보스, 연출 레퍼런스 이미지를 Google Flow 또는 다른 이미지/영상 생성 AI로 만들기 위한 인수인계 문서다.

목표는 포코팡의 원본 에셋을 복제하는 것이 아니다. 현재 엑셀 적/보스 데이터, 구현된 보스 스킬, 포트폴리오 방향성에 맞는 독창적인 아케이드 퍼즐 배틀 콘셉트 이미지를 만드는 것이 목표다.

참고 링크:

- Google Flow 제품 페이지: https://labs.google/flow/about
- Google Flow 도움말: https://support.google.com/labs/answer/16353334

## 제작 산출물

만들어야 할 산출물은 세 가지다.

1. 캐릭터 콘셉트 이미지
   - 일반 적은 1종당 전신 3/4 뷰 1장.
   - 보스는 1종당 기본 대기 이미지, 스킬 예고 이미지, 스킬 발동 이미지 3장.

2. 연출 레퍼런스 영상
   - Freeze, Stone, ColorSwap, Fever, Rainbow, Boss Spawn 각각 2-4초 영상.
   - 이 영상은 최종 Unity 리소스가 아니라 타이밍, 색감, 움직임 참고용이다.

3. Unity 적용 후보 에셋
   - 콘셉트 방향이 승인된 뒤에만 제작한다.
   - 투명 PNG, 통일된 캔버스 크기, 작은 화면에서도 읽히는 실루엣을 기준으로 한다.

## 아트 방향 규칙

- 모든 디자인은 독창적으로 만든다. 포코팡 캐릭터, UI, 몬스터, 에셋을 그대로 요청하지 않는다.
- 무섭거나 사실적인 몬스터보다 귀엽고 읽기 쉬운 아케이드 퍼즐 배틀 비율을 사용한다.
- 작은 디테일보다는 한눈에 구분되는 실루엣을 우선한다.
- 보스는 외형만 봐도 어떤 스킬을 쓰는지 보여야 한다.
- 일반 적은 작은 HP UI 옆에서도 알아볼 수 있을 정도로 단순해야 한다.
- 이미지 안에 텍스트, 로고, 워터마크, UI 라벨, 복잡한 배경을 넣지 않는다.
- 원본 이미지는 깔끔한 2D 모바일 게임 콘셉트 아트 스타일로 만든다.
- Flow/Veo 영상은 최종 영상 리소스가 아니라 Unity 코드 연출이나 VFX 제작을 위한 참고 자료로 사용한다.

## 공통 이미지 프롬프트 구조

각 이미지 프롬프트는 아래 구조를 따른다.

```text
2D 모바일 라인 연결 퍼즐 배틀 게임에 등장하는 독창적인 [일반 적 / 보스] 캐릭터 콘셉트 아트를 만들어줘.
캐릭터 이름: [이름].
역할: [작은 일반 적 / 보스 적].
스킬 콘셉트: [Freeze / Stone / ColorSwap].
실루엣: [한눈에 읽히는 형태 설명].
색상 팔레트: [주요 색 3-5개].
시각 모티프: [얼음 / 돌 / 프리즘].
스타일: 독창적인 캐주얼 모바일 게임 콘셉트 아트, 깔끔한 형태, 부드러운 2D 렌더링, 전신, 정면 3/4 뷰, 텍스트 없음, 로고 없음, 배경 없음.
분위기: 장난감처럼 수집하고 싶은 느낌, 귀엽고 읽기 쉬운 가족 친화적 스타일.
```

도구가 네거티브 프롬프트를 지원하면 아래 문장을 같이 사용한다.

```text
텍스트 없음, 로고 없음, 워터마크 없음, 사실적인 고어 없음, 공포 스타일 없음, 기존 게임 캐릭터 복제 없음, 포코팡 원본 스타일 복제 없음, 복잡한 배경 없음, 몸이 잘리지 않게, 과한 무기 장식 없음, 작은 화면에서 안 보이는 과도한 디테일 없음.
```

## Unity 에셋화 기준

생성 이미지를 콘셉트에서 Unity 에셋 후보로 승격할 때는 아래 기준을 따른다.

- 투명 PNG로 내보낸다.
- 권장 원본 캔버스:
  - 일반 적: `1024x1024`
  - 보스: `1536x1536` 또는 `2048x2048`
  - 연출 레퍼런스 스틸: `1920x1080`
- 캐릭터는 중앙에 배치하고 안전 여백을 둔다.
- 광원 방향과 선 굵기를 통일한다.
- AI 생성 아티팩트는 Unity에 넣기 전에 제거한다.
- 캐릭터 스프라이트 안에 텍스트나 UI를 굽지 않는다.
- 최종 임포트 에셋은 `md/`가 아니라 추후 Unity 아트 폴더에 저장한다.

추천 경로:

```text
Assets/PokoPuzzle/Art/Characters/RegularEnemies/
Assets/PokoPuzzle/Art/Characters/Bosses/
Assets/PokoPuzzle/Art/Effects/References/
```

`.meta` 파일은 수동으로 수정하지 않는다. Unity가 생성하게 둔다.

## 리뷰 체크리스트

생성 이미지를 채택하기 전에 아래를 확인한다.

- 작은 모바일 스케일에서도 캐릭터가 구분되는가?
- 보스의 스킬 콘셉트가 설명 없이 보이는가?
- 패밀리별 색상 팔레트가 서로 충분히 다른가?
- 포트폴리오에 사용할 만큼 독창적인가?
- 텍스트, 로고, 워터마크, 복제 캐릭터, 보호받는 게임 에셋이 없는가?
- 다른 AI나 아티스트가 이 프롬프트만 보고 같은 방향을 재현할 수 있는가?
